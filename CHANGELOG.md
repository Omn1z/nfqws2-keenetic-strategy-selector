  загружаются только новые.
### Изменено

- Выровнена высота кнопок действий в строке результатов (кнопка экспорта была
  ниже остальных).

## [v0.2.0] — 2026-05-28
### Добавлено
- **TG WS Proxy** — интегрирован проект [tg-ws-keenetic](https://github.com/Omn1z/tg-ws-keenetic),
  переписанный с Python на Go и встроенный в сервис отдельной вкладкой
  «TG WS Proxy». Это Telegram MTProto → WebSocket прокси прямо на роутере:
  клиенты в LAN ходят на `<роутер>:порт`, трафик идёт к Telegram по WSS
  (`kws*.web.telegram.org`) через указанный IP DC. Полный паритет с оригиналом:
  обфускация MTProto (AES-CTR, пере-шифрование без раскрытия полезной нагрузки),
  Fake-TLS-маскировка, пул «горячих» WS-соединений с прогревом, запасные пути —
  Cloudflare Worker / пул CF-доменов / прямой TCP к IP DC.
- Вкладка показывает статус и статистику (соединения, трафик, пул, ошибки),
  готовую `tg://` ссылку с кнопкой «Копировать», генерацию секрета и настройки
  (порт, DC-редиректы, Fake-TLS-домен, размер пула/буфера, CF-fallback,
  PROXY protocol, хост для ссылки).
- Прокси **выключен по умолчанию** (порт 1433), состояние сохраняется и
  восстанавливается при перезапуске сервиса; включение/выключение —
  переключателем во вкладке.

## [v0.1.5] — 2026-05-28
### Добавлено
- **Автоподбор стратегий** — режим прогона, который сам перебирает встроенный
  каталог кандидатов (≈30 проверенных комбинаций desync) с учётом выбранных
  блобов и целей списка (домены/IP, в т.ч. импортированных из GeoSite/GeoIP).
  Перед перебором делается базовый замер без обхода, поэтому проверяются только
  реально заблокированные цели; рабочие стратегии сохраняются и сортируются по
  скорости.
- **BlockCheck** — отдельная вкладка: проверка блокировки целей списка без
  обхода (тестовое соединение исключается из основного сервиса nfqws2 и не
  обходится). Вердикт по каждой цели: доступен / таймаут / RST / обрыв на
  ~16 КБ / DNS / отказ.
- Выбор стратегий и блобов для прогона — списками чекбоксов с кнопкой
  «Выбрать все» (умный тоггл: выделяет невыбранное, либо снимает все галочки).
### Изменено
- Таблица результатов прогона обновляется по мере проверки — каждая
  проверенная стратегия сразу попадает в результаты.
- Ручное добавление стратегии теперь валидируется (как в nfqws2-keenetic
  запрещён `--new`; плюс проверка имён методов `--lua-desync`/`--dpi-desync`).

## [v0.1.3] — 2026-05-28
### Добавлено
- **Авторизация веб-интерфейса** по логину/паролю — проверка против системного
  аккаунта роутера (`/opt/etc/passwd` / `/etc/shadow`, crypt), как у
  nfqws-keenetic-web. Сессия в cookie, выход, переключатель в настройках
  (по умолчанию включена). Аварийное отключение через `N2S_NOAUTH=1`.
- **Выбор блобов для прогона** — в настройках прогона можно отметить блобы
  (системные и пользовательские); выбранные загружаются во все тестируемые
  стратегии (`--blob=имя:@путь`), чтобы стратегии могли на них ссылаться.
- **GeoSite/GeoIP** — загрузка файла `geosite.dat`/`geoip.dat` (формат v2ray)
  или текстового списка, просмотр категорий и импорт категории (с лимитом) в
  тестовый список.
